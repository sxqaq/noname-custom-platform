# 阶段 0–4（五个阶段）验收说明

本项目定位是网页优先的朋友联机平台。账号、匹配、排位、反作弊、公开审核、举报、版权工单和大规模运营不在当前范围。

## 阶段 0：技术验证

- 已验证浏览器可通过 WebSocket 消息驱动权威对局。
- 无名杀上游直接导入 Node.js 仍依赖浏览器环境，因此采用独立无 DOM 规则内核，并把无名杀固定源码作为内容兼容基线。
- 所有随机行为来自种子随机源；快照恢复和相同命令序列可稳定复现。

## 阶段 1：最小可玩联机

- 游客身份、稳定会话令牌、大厅、创建/加入/准备房间和 2–8 人身份局。
- 服务端权威选将、完整固定标准包、断线恢复原座位与私有视图、基础回放。
- 在线玩家超时和离线玩家可由服务端 AI 托管，托管命令同样进入回放。

## 阶段 2：自定义武将

- 武将基本信息、技能 DSL、预览和使用同一规则内核的单机测试。
- JSON 扩展包导入导出、发布前校验与测试、不可覆盖版本和 SHA-256 房间内容锁。
- 房间按明确的扩展 ID 与版本启用内容。

## 阶段 3：丰富创作

- 节点技能编辑器、自定义卡牌、牌堆、模式和胜利条件。
- `@sgs/script-sdk` 把高级 TypeScript 创作编译为声明式 DSL；服务器不执行作者脚本。
- 扩展自动测试、在线内容列表、分享 ID 和只读分享接口。

## 阶段 4：朋友平台跨端交付

- PWA manifest、Service Worker 应用外壳缓存、响应式网页和统一平台能力接口。
- 生产环境由同一服务提供网页、HTTP API 与 WebSocket；提供 Docker 单机部署和持久化扩展库。
- 业务层不依赖浏览器专属存储或文件 API，为未来 Capacitor/Tauri 外壳保留适配边界。

当前朋友平台范围不交付 Capacitor/Tauri 安装包，也不实现微服务横向扩容；这是明确裁剪，不是网页架构阻塞。若以后需要原生文件、音频后台恢复或多实例部署，可在现有平台接口和权威服务边界上继续扩展。

详细标准内容覆盖见 [standard-content-progress.md](standard-content-progress.md)。
