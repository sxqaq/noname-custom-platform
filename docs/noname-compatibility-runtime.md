# 无名杀兼容运行时

目标不是把无名杀的房主浏览器权威联机原样搬进项目，而是在服务端复用其成熟的技能声明、事件模型和扩展 API，同时保持本项目的权威命令、快照、回放和局域网分发。

## 固定上游

- 仓库：`libnoname/noname`
- 子模块：`vendor/noname`
- 固定提交：`632d2d3c8da2893466a8c440a18861c9ed49813d`
- 许可证：GPL-3.0；再分发必须保留许可证、来源和对应源码。

## 已验证

`@sgs/noname-adapter` 可以读取固定子模块中的真实 `character/*/skill.js`，移除模块装载层后在 Node `vm` 上下文中建立技能对象，而不启动浏览器、DOM 或上游房主客户端。

- 标准包在无 DOM Node 环境成功加载。
- 真实技能 `stdqingjiao` 的异步 `content` 已由宿主 `player` 代理执行。
- 真实技能 `ganglie.check` 中的 `Math.random` 已替换为可快照、可恢复的种子随机源。
- 真实技能 `fanjian` 的选择花色、获得手牌两段异步操作已转换为可序列化 `NonameInteractionRequest`，并由指定玩家的外部命令继续执行。
- 两段交互响应形成日志；重新执行技能并注入相同日志会得到相同结果，响应者身份不匹配会被拒绝。
- `ReplayableNonameExecution` 可以保存输入日志和当前待选择请求，销毁执行实例后重放到完全相同的异步位置。
- 高级作者代码可以在一次性 Worker 与内层 VM 中执行；结构化输入、确定性随机、内存上限、执行超时和强制终止已经验证，系统 API、动态代码生成和系统时间不可用。
- `.sgspack` 已能声明 `noname-compat/v1`、固定上游提交、权限和资源限制，并在清单中锁定编译源码 SHA-256；局域网房主公开相同能力标识。
- 双 WebSocket 客户端端到端测试已验证：房主发布高级 Mod、另一台设备加入并准备、服务器执行 `roomStart/afterCommand` 钩子，并向双方发送一致的裁剪状态；非法运行时效果会被 Schema 拒绝。
- 开局期间的运行时错误会把房间从 `playing` 原子回滚到 `waiting`；广播和离线自动化只访问已经实际建立的权威对局。
- `player-choice` 权限已接入目标、卡牌、选项、数字、花色五类交互；请求和响应均由服务器校验，待选择状态支持快照、断线重连、离线 AI、链式继续和无代码回放。
- `defineNonameSkillRuntime()` 已把 `chooseBool/chooseControl/chooseTarget/chooseCard` 接入服务器权威规则事件：异步技能可暂停当前事件、跨快照恢复、继续后续扩展，并由普通房间回放还原已校验结果。
- `GameManager` 已在开局、玩家命令和离线 AI 命令后运行隔离钩子；返回效果经现有规则 Schema 校验，再由权威引擎原子应用。
- `afterCommand` 输入包含完整只读命令、核心事件日志、实际操作者和首个目标；高级效果的 `self/selected` 由这些权威 ID 映射，并纳入输入哈希和确定性回放校验。
- 核心命令或兼容 Mod 任一失败时，游戏状态、随机状态、命令日志和 Mod 检查点一起回滚；基础回放使用已锁定钩子输出，不重新执行作者源码。
- 武将包 ID 和实际路径均限制在固定上游目录内。
- 固定提交中的 26 个官方武将技能包均进入无 DOM 批量加载回归测试；`offline` 的 `dedent` 与 `xianding` 的空卡牌声明导入由白名单加载期 shim 处理，未知额外导入仍被拒绝。
- CLI 可生成真实 API 调用矩阵与指定技能迁移骨架；当前 602 种调用形式中仍有 556 种未映射，因此“全部包可加载”不代表“全部技能可直接执行”。详见 [无名杀 API 兼容矩阵](noname-api-matrix.md)。

这里的 `vm` 只用于加载仓库固定、受信任的 GPL 上游源码，不是第三方 Mod 的安全边界。第三方代码必须放入独立 Worker/子进程，并增加 CPU、内存、文件和网络权限限制。

## 下一步宿主协议

1. 继续扩充 `player.choose*`、`game.*`、`get.*` 的显式能力表。
2. 在已有布尔、选项、目标、卡牌选择上增加判定、排列、分配、多目标和受限自定义对话框。
3. 将玩家、卡牌、区域和事件代理映射到权威状态，不把真实状态对象交给客户端。
4. 在稳定检查点保存随机状态、待选择请求和响应日志；恢复时从检查点重放技能，而不是序列化 JavaScript 调用栈。
5. 按真实技能测试集标记 `direct`、`shimmed`、`migrated`、`unsupported` 四级兼容性。
6. 扩展 `.sgspack` 清单，声明兼容 API、权限、上游版本和入口模块。

## 高级扩展清单

高级扩展在普通内容数据之外增加 `runtime`：

```json
{
  "kind": "noname-compat",
  "apiVersion": "noname-compat/v1",
  "upstreamCommit": "632d2d3c8da2893466a8c440a18861c9ed49813d",
  "source": "(input) => ({})",
  "permissions": ["game-state", "player-choice", "deterministic-random"],
  "limits": { "timeoutMs": 500, "memoryMb": 32 }
}
```

源码是 TypeScript 构建后的单入口 JavaScript，最大 1 MiB。清单权限只描述可申请能力，宿主仍按每次调用实际授予最小能力。扩展包内容哈希覆盖完整源码；`.sgspack` 另保存源码哈希，任何源码或权限篡改都会在导入时失败。

## 完成标准

兼容运行时只有同时满足以下条件才可进入房间默认路径：

- 相同种子和命令序列产生相同快照；
- 技能等待玩家输入时可以重启、恢复和继续；
- Mod 无法直接访问 DOM、文件、网络、系统时间或系统随机数；
- 房主服务器执行规则，客户端只提交经过校验的选择；
- 插件错误只终止对应房间或插件实例，不拖垮宿主进程。
