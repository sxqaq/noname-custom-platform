# 自由创作系统说明

本系统的最终形态是：每位安装者都拥有完整的网页工坊、权威规则主机和局域网内容节点。作者可以用表单、节点图或 TypeScript SDK 创建武将；朋友加入房主网页后使用房主锁定的同一扩展版本、资源、随机种子和规则快照游玩。

## 创作层次

1. 表单层适合基本资料、卡面、卡牌、牌堆和常见技能效果。
2. 节点图层使用稳定节点 ID、入口和显式连线，提供发布前结构诊断与运行轨迹。
3. TypeScript SDK 适合复用函数和工程化组织，但只能生成规则 IR；沙箱和联机服务器都不允许作者代码越过 IR 直接操作对局。

## 一致性边界

- 权威状态、身份、选择合法性、随机数和胜负都在无 DOM 服务端引擎中计算。
- 客户端只提交命令和渲染裁剪后的视图。
- 扩展版本按 SHA-256 锁定；快照、AI、断线恢复和回放使用相同规则内容。
- `.sgspack` 包含内容清单、资源和精确依赖，导入时逐项校验哈希。

## 局域网工作流

1. 房主在本机工坊创作并发布不可覆盖版本。
2. 内容库可下载 `.sgspack`，也可复制房主分享链接。
3. 另一位安装者把文件导入自己的本地主机，即可独立成为房主。
4. 创建房间时选择扩展版本，房间锁定其 ID、版本和哈希。
5. 旧版本可以并存，用于旧回放或回退；存在依赖时禁止误删基础版本。

## 当前安全模型

平台面向互相信任的朋友房，不提供账号、排位和反作弊。TypeScript 构建沙箱禁止系统导入、文件、网络、进程、时间、动态导入和 `Math.random`，并设置超时；联机端仅执行有配额的声明式 IR。静态图片按内容哈希存储，上传和安装均受尺寸及完整性限制。
