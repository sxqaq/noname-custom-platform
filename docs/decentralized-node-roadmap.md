# 去中心化节点与开放插件路线图

> 状态：阶段 A–E 已完成并通过自动测试、双客户端 E2E、10,000 局模拟和 Windows 安装器构建；阶段 F 按产品范围暂缓。

## 产品目标

每个完整安装版都是一个独立节点。用户既能加入其他节点，也能在自己的设备上启动权威房间、选择本地扩展并把该局锁定的图片、卡面和规则资源提供给其他玩家。当前交付范围是无需外部服务的本机与局域网模式。

房间仍采用“单主机权威”模型。去中心化指主机可由任意安装者运行，不表示多个客户端共同修改规则状态。

## 不可破坏的系统约束

1. 身份、隐藏牌、随机数、合法性和胜负只由当前房主节点决定。
2. 同一种子、扩展哈希和玩家命令必须产生相同快照。
3. 所有等待玩家输入的插件步骤都必须可序列化，以支持断线、托管和回放。
4. 房主可以分发静态资源和声明式客户端界面，但不能向玩家浏览器注入任意 JavaScript。
5. 高级插件不能直接访问 Node.js、网络、文件、系统时间或非种子随机源。
6. 扩展发布版本不可覆盖；房间和回放保存完整依赖锁。
7. 房主是朋友房信任边界。竞技反作弊不属于本路线图。

## 阶段与验收门

### A. 节点契约与可嵌入 Host Runtime

- 服务器可由 CLI、测试、桌面 sidecar 或 Docker 以同一 API 启动。
- 监听地址、端口、数据目录、网页目录和节点名称均可配置。
- 提供节点信息、能力和健康接口。
- 关闭运行时必须释放端口、WebSocket 和定时任务。

验收：测试可以在随机端口启动两个隔离节点，分别创建内容库并完整关闭。

### B. 局域网联机

- 支持局域网监听、手动地址连接和同源网页入口。
- 节点持久化身份密钥，邀请中包含节点指纹。
- 使用 mDNS 广播 `_noname-sgs._tcp.local`，并提供关闭发现的设置。
- 展示可连接地址、端口、防火墙状态和诊断结果。

验收：两台同一局域网设备在断网情况下自动发现、加入、重连并完成一局。

### C. 内容寻址资源同步

- 扩展包升级到包含资源清单、作者、许可证和引擎约束的新 Schema。
- 图片上传后进行解码校验、尺寸限制、裁剪配置、缩略图和哈希存储。
- 节点提供内容清单、按哈希资源下载、`.sgspack` 打包安装和缓存接口。
- 客户端只加载安全 MIME 与声明式卡面配置。

验收：只有房主拥有自定义武将立绘和卡面，加入者能按哈希下载、校验、缓存并在重连后复用。

### D. 开放插件 SDK

- 作者使用 TypeScript SDK 定义事件、条件、选择和效果。
- 编译器生成确定性规则 IR；服务器执行 IR，不执行作者源码。
- 所有宿主能力都有显式版本和配额。
- 提供 CLI、热重载、单元测试、模拟对局和兼容检查。

验收：简单触发技、多段主动技、判定/响应复杂技能三个示例均通过联机、AI、快照和回放测试。

### E. 完整安装版

- 首个目标为 Windows 安装版，同时保持 macOS/Linux 打包结构。
- 安装版包含网页、Host Runtime、规则引擎、内容库和节点控制面板。
- 支持“作为主机”“加入节点”“打开数据目录”“查看诊断”“导入扩展”。
- 默认仅本机监听；用户明确开启局域网后才绑定 LAN 地址并触发防火墙提示。

验收：干净 Windows 环境无需 Node.js 即可安装、开房、加入、卸载；用户数据与程序目录分离。

### F. 互联网联机

按当前产品范围暂缓，不是阶段 A–E 的交付依赖。将来恢复时仍遵循以下边界：

- 第一层支持公网 IP、IPv6、端口映射和自定义域名。
- 第二层抽象可替换信令与 ICE/WebRTC 传输。
- 第三层允许自建 TURN 或字节中继，服务不成为规则权威。

验收：两台不同 NAT 网络的设备通过邀请码连接；关闭默认信令后可以切换到自建服务。

## 最低交付线

阶段 A–E 是当前最低交付线：可安装、可开房、可局域网联机、可同步本地资源、可使用开放代码插件。阶段 F 已按产品决定暂缓；恢复时不得用中央房间服务器替代本地权威节点。
